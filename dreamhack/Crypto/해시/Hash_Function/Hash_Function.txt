암호학적 해시 함수(Cryptographic Hash Function) :
    암호학적 해시 함수(Cryptographic Hash Function)는 다음 성질을 만족하는 해시 함수를 말합니다.

        1) 제 1 역상 저항성(Preimage Resistance):
            암호학적 해시 함수 H에 대해 y가 주어졌을 때 H(x) = y를 만족하는 x를 찾는 것이 어렵다.
            이는 함수가 일방향 함수여야 함을 의미합니다.

            해시 값이 주어졌을 떄, 그 해시 값을 생성하는 입력값을 알아내기가 불가능하다는 특성.
            x를 계산하기 어려워야 함.

        2) 제 2 역상 저항성(Second Preimage Resistance):
            암호학적 해시 함수 H에 대해 x가 주어졌을 때 x != x′, H(x) = H(x′)을 만족하는 x'을 찾는 것이 어렵다.

            어떤 입력 값과 동일한 해시 값 (결과 값)을 가지는 다른 입력 값을 찾을 수 없어야 한다는 특성.
            해시 값을 알고 있을 떄 원래 값을 찾아내는 것이 어려워야함.
            
        3) 충돌 저항성(Collision Resistance): 
            암호학적 해시 함수 HH에 대해 x != x′, H(x) = H(x′)을 만족하는 x, x′을 찾는 것이 어렵다.

            해시 값 (결과 값)이 이 같은 입력 값 두 개를 찾을 수 없다는 특성.
            해시 값이 무엇이든 (그 값을 알든 모르든) 특정하게 같은 해시 값을 생성하는 값 두개를 찾는 과정이 어려워야함.

    추가로, 현대에는 눈사태 효과(Avalanche Effect)를 이상적인 해시 함수의 조건 중 하나로 보기도 합니다.
    이 성질은 대칭키 암호 시스템의 확산과 비슷하게 입력에 조그만 변화가 발생하면, 해시값에도 큰 변화가 발생하는 성질입니다.


암호학적 해시 함수의 사용 :
    암호학적 해시 함수는 어떤 통신의 무결성을 보이기 위해 사용될 수 있습니다.
    수신한 데이터가 원본과 같을 때, 이 데이터는 무결하다고 하며, 데이터가 무결하게 송수신되는 성질을 통신의 무결성(Integrity)이라고 합니다.
    암호학적 해시 함수는 충돌 저항성을 만족하므로, 임의 데이터의 해시 값은 그 데이터의 고유값이라고 볼 수 있습니다.
    따라서 송신자가 데이터와 함께 데이터의 해시 값을 보내면, 수신자는 받은 데이터로부터 해시값을 생성하고,
    이를 송신자가 보낸 해시값과 비교하여 받은 데이터가 변조되지 않았는지 확인할 수 있습니다.

    또한, 암호학적 해시 함수는 민감한 데이터를 보관할 때 사용될 수 있습니다.
    웹 서버의 데이터베이스에서 사용자의 비밀번호를 평문으로 저장하면 관리자가 비밀번호를 유출하거나,
    해킹 사고로 사용자의 계정 정보를 모두 도난당할 위험이 있습니다.
    암호학적 해시 함수는 일방향 함수이므로 비밀번호를 해시 값으로 저장하면 데이터베이스가 유출돼도 비밀번호의 원본이 알려질 가능성이 거의 없습니다.
    사용자를 인증할 때는 사용자가 입력한 비밀번호의 해시 값을 저장된 해시 값과 비교합니다.
